<Start: SRS-163769-HeNBFM-100, Requirement>
HeNB shall support sending of all alarms (including clears) and events over the TR069 interface as informs.
The alarms will be translated into informs and will contain all the following fields.
Alarm notification as raised in eNB via SNMP	Description	Mapping for alarm informs over TR069
alarmLastChangeCounter,	Counter incremented with each alarm change.
introduced for NB for alarm mgt and synchronization, not an alarm field itself	alarmLastChangeCounter
 managedObjectInstance,	MO Identity	managedObjectInstance
alarmTime,	The time at which the fault occurred (mandatory). It should be in the format of  Local Time + UTC Offset [Can be Positive or Negative], i.e. YYYYMMDDThhmmss+/-hhmm, for example: 20080827T142403-0100.	alarmTime
alarmType,	The type of the alarm: Communication, quality of service, processing error, equipment, or environment-related.
The alarm types are defined in the CCITT recommendation X.733, therefore only 'communicationsAlarm', 'objectDeletion', 'qualityofServiceAlarm', 'processingErrorAlarm', 'equipmentAlarm',  'environmentalAlarm', 'miscellaneous', and 'unknown' from ExtendedX721EventType are used.	alarmType
alarmSeverity,	None, warning, minor major, critical, cleared, unknown
	alarmSeverity
alarmId,	Identification of the alarm - alarm ID in range 400 0001 to 500 0000	alarmId
alarmAdditionalInformation,	text string providing additional information	alarmAdditionalInformation
alarmMonitoredAttribute,	text string describing MO 	alarmMonitoredAttribute
alarmOperationalState,	Operational state of the managed object induced by this alarm (not necessarily its current operational state).	
alarmAvailabilityStatus,	Availability status of the managed object 	alarmAvailabilityStatus
         numberOfFilteredAlarmsSinceLastReporting,		numberOfFilteredAlarmsSinceLastReporting,
lastFilteringTime,	UTC time of last filtered alarm	lastFilteringTime
lastFilteringType,	type of last filtered alarm (none/begin/end/pingpong)	lastFilteringType
alarmSpecificProblem,	text string with further detail to X-733 Probable Cause value, e.g., used for external alarms	alarmSpecificProblem
alarmFdn,	Fully Distinguished Name of the managed object where this alarm occurred. The syntax is defined in SID eNodeB Configuration Management. If the object does not have an FDN, the eNB's FDN is shown. The value must not be the empty string	alarmFdn
alarmNature	ADAC or ADMC	alarmNature
