<Start: SRS-162298-HeNBStrtUp-800, Requirement>
When starting up, if the HeNB has previously registered with HDM (registration status is "true"), the HeNB shall initiate a TR-069 session by sending a BOOT Inform request to the assigned HDM using the provisioned username and password, and shall start the registration timer (Note: See SRS-162298-HeNBStrtUp-880 for details).
If the timer expires the HeNB shall take the actions specified in SRS-162298-HeNBStrtUp-880.
The format of the Inform message shall be as follows:
 *	Device id
  *	Manufacturer = DeviceInfo::Manufacturer
  *	OUI = DeviceInfo::ManufacturerOUI
  *	ProductClass = DeviceInfo::ProductClass
  *	Serial Number = A truncated form of DeviceInfo::SerialNumber, containing the 12 numerical characters as specified in FRS 162839.
 *	Event = Boot 
 *	MaxEnvelopes = 1
 *	CurrentTime = The current date & time known by the HeNB
 *	ParameterList shall contain the following
 *	DeviceInfo::SoftwareVersion
 *	Enb::mimVersion
Note: The username & password shall be obtained from ManagementServer::Username & ManagementServer::Username  respectively
Note: If the HeNB does not have configuration data loaded, Enb::mimVersion will be set to "null".
FRS Requirement: FRS-162298-Req-R7          
	Affected DPM Area: HeNB-OAM, SCMS, Test          
	Release: SCLR1.1          
	Network Element: HeNB-OAM, SCMS, Test          
<End: SRS-162298-HeNBStrtUp-800>
